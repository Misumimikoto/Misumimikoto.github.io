数据结构之线性表

本文是关于数据结构中线性表的一些总结和自我理解，便于自己以后进行知识的复盘和总结。

线性表的概念及其基本操作

线性表本身是由n个元素组成的有限长序列，其中最主要的是**节点按下标次序具有逻辑关系**。

结点的前趋和后继：若给定一个线性表，则结点a(i - 1)称为结点a的前趋结点。a(i + 1)称为结点a的后继节点。因此，a0无前趋而a(n - 1)无后继

线性表的基本操作：初始化线性表、置空表、求表长、取元素、求元素的前趋和后继、*i号下标之前插入结点x**、删除 i号下标元素***等。

线性表的顺序物理存储结构

1. 物理结构及其定义

物理结构：可以用大于或等于n的一维数组实现存储

顺序表的C语言实现：

typedef 数据元素类型 ElemTp
#define MAX_N 数组最大容量  

typedef struct {
    ElemTp elem[MAX_N];//实际装入存储的数据
    /*或者使用 *elem 动态分配存储空间*/
    int max_size;//顺序表最大容量
    int n;//线性表实际长度
}Sqlist;

2. 算法实现（以两个有序顺序表的归并算法为例）

//两个升序顺序表la和lb归并为lc，使lc保持升序

/**************************
纯C不支持bool类型 此处仅作演示
**************************/

bool merge(Sqlist &la, Sqlist &lb, Sqlist &lc){
    //归并超出最大容量，失败
    if(la.n + lb.n > lc.max_size) return false;
    int i = j = k = 0;
    lc.n = la.n + lb . n;
    while(i < la.n && j < lb.n){
        //按照升序归并
        if(la.elem[i] < lb.elem[j]) lc.elem[k++] = la.elem[i++];
        else lc.elem[k++] = lb.elem[j++];
    }
    //若其中一个表还没有归并完成的话就继续
    while(i < la.n) lc.elem[k++] = la.elem[i++];
    while(j < lb.n) lc.elem[k++] = lb.elem[j++];
    //归并成功
    return true;
}

//时间复杂度为O(m + n)

3. 优缺点简单比较

优点

  1. 数组元素下标与节点序号一致，存取元素很方便（有利于随机存取）。

缺点

  1. 插入和删除等操作变得更加复杂；
  2. 表的长度不够灵活，可能会导致不够用或者空间浪费

线性表的链式物理存储结构

1. 链表存储结构

本质上就是C语言中的链表实现

不带附加头节点，链表头插入、删除等操作会导致头指针的改变，增加了处理难度

此处使用带有附加头结点的单向链表，此时头结点中不存数据。使很多算法得到简化。

还有许多结构例如单向循环列表、双向链表、双向循环列表等

2. 单向链表的数据结构类型和C语言实现

数据结构如下所示：

typedef struct node{
    ElemTp data;
    struct node *next;//后继指针
}LNode, *Linklist;

单向链表的建立：

先入先出法

//伪代码，不代表实际编译，但可以反映思想

createnode(h);//建立头结点
last = h;//指向本次操作的最后一个结点
while(1){
    getdata();//将数据存入
    createnode(p);//新建结点并分配空间
    p->data = getdata();//赋值
    last->next = p;//连接
    last = p;//指向当前的最后结点
}
last->next = NULL;//置空
return h;//返回头结点

先入后出法

createnode(h);//建立头结点
h->next = NULL;//空链表
while(1){
    getdata();//将数据存入
    createnode(p);//新建结点并分配空间
    p->next = h->next;//新插入结点的下一个结点是原来头结点的下一个结点，实现了先入后出
    h->next = p;//现在头结点的下一个成了最新建立的结点
}
return h;
